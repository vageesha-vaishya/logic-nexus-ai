# Multi-Tenant Quotation Branding System

## Overview
The Multi-Tenant Branding System allows each tenant to configure their own branding assets (logo, colors, texts) which are dynamically applied to generated Quotation PDFs. This system replaces the legacy hardcoded or single-tenant branding approach while maintaining backward compatibility.

## Architecture

### Database Schema
The `quotation_configuration` table has been extended with a `branding_settings` JSONB column.

```sql
ALTER TABLE quotation_configuration 
ADD COLUMN branding_settings JSONB DEFAULT '{}'::jsonb;
```

**Branding Settings Structure:**
```json
{
  "logo_url": "https://...",
  "primary_color": "#FF0000",
  "secondary_color": "#00FF00",
  "accent_color": "#0000FF",
  "company_name": "My Logistics Co",
  "company_address": "123 Main St...",
  "header_text": "Global Shipping",
  "sub_header_text": "Reliable & Fast",
  "footer_text": "Generated by System",
  "disclaimer_text": "Standard terms apply...",
  "font_family": "Helvetica"
}
```

### Storage
- **Bucket**: `organization-assets`
- **Path**: `{tenant_id}/branding/logo-{timestamp}.{ext}`
- **Access**: Public read, Authenticated upload (RLS restricted to tenant's folder).

### PDF Generation Flow
1. **Frontend**: User configures branding in `QuotationSettingsPanel` -> `BrandingSettingsForm`.
2. **Backend**: Settings saved to `quotation_configuration`.
3. **Edge Function (`generate-quote-pdf`)**:
   - Fetches `quotation_configuration` for the quote's tenant.
   - Extracts `branding_settings`.
   - Fallback: If no branding settings, checks legacy `tenant_branding` table.
   - Fetches logo image from URL and converts to Base64 (chunked processing).
   - Injects branding data into `SafeContext`.
4. **Renderer**: `renderer.ts` applies styles and texts to the PDF definition (pdfmake).

## API Specifications

### Update Configuration
- **Method**: `UPSERT` on `quotation_configuration` table.
- **RLS**: Tenants can only update their own row.

### File Upload
- **Bucket**: `organization-assets`
- **Method**: Standard Supabase Storage upload.

### PDF Generation
- **Endpoint**: `/functions/v1/generate-quote-pdf`
- **Auth**: Service Role Key or User JWT.
- **Payload**:
  ```json
  {
    "quoteId": "uuid",
    "versionId": "uuid" (optional)
  }
  ```

## Deployment & Testing

### Deployment
1. Apply migration: `supabase/migrations/20260306000001_add_branding_configuration.sql`
2. Deploy Edge Function: `supabase functions deploy generate-quote-pdf`
3. Set Secrets: Ensure `SUPABASE_SERVICE_ROLE_KEY` is available to the function.

### Testing
- **Script**: `scripts/test_new_branding_flow.cjs`
- **Usage**: `node --env-file=.env scripts/test_new_branding_flow.cjs`
- **Bypass Key**: For automated testing without a valid user JWT, set `TEST_BYPASS_KEY` secret and send `x-bypass-key` header.

## Backward Compatibility
The system checks for branding in the following order:
1. `quotation_configuration.branding_settings`
2. `tenant_branding` table (Legacy)
3. Default System Branding (Nexus Logistics)
