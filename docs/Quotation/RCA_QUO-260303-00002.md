# Root Cause Analysis: QUO-260303-00002 PDF Rendering Issue

## Incident Overview
**Issue:** Quotation QUO-260303-00002, containing multiple transport modes and legs, failed to display the expected "Standard Multi-Modal" template. Instead, it rendered a generic template missing the transport leg breakdown.
**Date:** 2026-03-06
**Impact:** Customers receiving multi-leg quotes could not see the detailed transport chain (Road -> Ocean -> Road), leading to ambiguity.

## Root Cause
The investigation identified three contributing factors:

1.  **Missing Template Section**: The "Standard Multi-Modal" template configuration in the database was missing the `multi_modal_details` section definition. Even if selected, the renderer would not know to render the leg breakdown.
2.  **Strict Template Selection Logic**: The previous template selection algorithm relied heavily on explicit `templateId` or simple default fallbacks. It lacked a robust heuristic to automatically detect multi-modal characteristics (multiple legs, mixed modes) when no template was explicitly assigned.
3.  **Data Mapping Gaps**: The PDF generation service's data mapper (`createRawData`) did not fully propagate all leg-level details (specifically `transport_mode`, `pol`, `pod`, `carrier`) required for the multi-modal table.

## Resolution
The following corrective actions were taken:

1.  **Database Migration**: Created `20260306120000_update_multimodal_template.sql` to inject the `multi_modal_details` section into the "Standard Multi-Modal" template JSON structure.
2.  **Algorithm Enhancement**: Updated `generate-quote-pdf/index.ts` to include a heuristic detection step. It now scans for >1 leg or Ocean/Air modes and automatically attempts to load the "Standard Multi-Modal" template.
3.  **Renderer Implementation**: Implemented `renderMultiModalDetails` in `renderer.ts` to dynamically generate a table of transport legs, handling pagination and fallback values (e.g., using `transport_mode` if `mode` is null).
4.  **Data Mapping Fix**: Enhanced `context.ts` and `index.ts` to ensure all leg properties are correctly mapped from the database to the PDF context.

## Verification
-   **Unit Testing**: A test suite (`scripts/test_pdf_generation_suite.cjs`) verified that QUO-260303-00002 is correctly identified as multi-modal, while simple quotes are not.
-   **Rendering Verification**: A rendering test (`scripts/verify_multimodal_pdf.cjs`) confirmed that the `multi_modal_details` section renders correctly with the mock data from the quote.

## Prevention
-   **Documentation**: Created `docs/Quotation/MULTI_MODAL_TEMPLATE_LOGIC.md` to document the selection criteria.
-   **Regression Testing**: The new test suite can be used to validate future changes to the PDF generation logic.
