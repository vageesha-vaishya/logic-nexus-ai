# Usage Guidelines

## Do and Don't

### Do

- Use `primary` for highest-priority page actions.
- Use `secondary` for supporting actions.
- Use `danger` only for destructive workflows.
- Preserve keyboard focus order for all interactive controls.

### Don't

- Don't use `danger` as a default visual style.
- Don't remove `aria-label` from icon-only interactions.
- Don't hardcode spacing outside token values.
- Don't ship components without Storybook play coverage.
